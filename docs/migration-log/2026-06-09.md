# Migration Log - 2026-06-09

## Mobile UI Refinement After Product Vision Convergence

- Compressed the Today health overview and changed Today priorities into a repository-provided list rendered by the frontend, with hydration shown as count-based progress instead of a fixed ml-only target.
- Removed Today quick-record shortcuts and campus-service priority exposure so Today stays focused on proactive reminders, medication, hydration, todos, and advice.
- Added selectable water units for daily record water input while preserving `ml` as the default stored unit.
- Moved Record quick operations above the timeline, removed the lower mobile calendar overview, made the Today record overview full width, and flattened nested Record surfaces.
- Rebuilt Record today overview from dashboard summary/timeline data, removed hardcoded sleep/mood/report-action rows, and rendered quick record entries as a divider grid instead of nested small cards.
- Narrowed Record active mobile entry types to symptoms, hydration, meal, sleep placeholder, and medication, while keeping legacy/non-MVP record kinds editable only when an existing record already uses them.
- Removed the old voice/photo/template quick-operation panel and its unused l10n strings from the active Record dashboard.
- Changed Report to show the current static snapshot on entry and refresh only after the user taps generate or sync.
- Removed the Report privacy settings block; privacy remains owned by Mine/Settings while Report keeps summaries, trends, export, and reference notice.
- Flattened Mine campus services from nested grid cards into a divider-based two-column service grid.
- Removed the outer Mine campus-services panel so the grid is organized directly by dividers under the section title.
- Narrowed Mine profile editing and summary to basic MVP fields only: birth date, height, blood type, unit system, and onboarding state. Pregnancy/lactation/special-group context remains deferred for medicine safety instead of being surfaced as a standalone Mine profile flow.
- Updated ARB strings and regenerated Flutter localizations.
