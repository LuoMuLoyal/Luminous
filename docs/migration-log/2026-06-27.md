# Migration Log - 2026-06-27

## Today 推荐行后端随机化接入

将 Today 页面的推荐行从本地硬编码改为后端驱动，支持随机推荐与相邻去重。

### What changed

**Lucent backend**

- 新增 `GET /api/v1/user/today-analysis/recommendations`。
- 新增 `TodayRecommendationsService`：从本地化推荐池中随机返回 3 条，支持 `exclude` 去重与中英文切换。
- 新增 `TodayRecommendationResponseDto`。
- `TodayAnalysisController` / `TodayAnalysisModule` 注册新 endpoint 与服务。
- 导出更新后的 `docs/openapi.json`。
- `today-analysis.controller.spec.ts` 补充 `TodayRecommendationsService` provider，修复 controller 测试依赖缺失。

**Luminous frontend**

- Regenerated `packages/lucent_openapi`。
- 新增 `lib/features/today/domain/entities/today_recommendation.dart`。
- 新增 `lib/features/today/data/datasources/today_recommendations_remote_data_source.dart`。
- 新增 `todayRecommendationsProvider`：管理推荐列表，refresh 时传递当前 ids 去重。
- `TodayRecommendationSection` 改为 `ConsumerWidget`，从 provider 异步加载，支持 loading/error/空状态。
- 新增 `lucentTodayAnalysisApiProvider`。

### Verification

- `cd Lucent && pnpm lint:check && pnpm build && pnpm test:ci && pnpm export:openapi` ✅
- `cd Luminous && flutter analyze` ✅
- `cd Luminous && flutter test` ✅

## Medicine Safety Follow-Up：过敏严重度推断、跨语言重复成分、中文药相互作用

修复 Medicine 安全卡片的三个风险检查缺口。

### What changed

**Lucent backend**

- `src/modules/medicines/dto/medicine-detail.dto.ts`
  - `CnMedicineDetailDto` 新增 `drugbankIds?: string[] | null`，把 V2 数据源里的 DrugBank 映射暴露给前端。
- `src/modules/medicines/sources/cn-medicines.service.ts`
  - `getDetail` 映射时把 `row.drugbankIds` JSONB 解析为字符串数组写入 DTO。
- 重新导出 `docs/openapi.json`。

**Luminous frontend**

- 新增 `lib/features/medicine/domain/services/allergy_severity_helper.dart`
  - `inferredAllergySeverity(...)`：当过敏 `severity` 为空/unknown 但 `reaction` 包含 anaphylaxis / 过敏性休克 / 严重过敏 / 休克 等关键词时，推断为 `severe`。
  - `isSevereAllergy(...)`：基于推断结果判断是否为严重过敏。
- `lib/features/medicine/domain/services/red_flag_evaluator.dart`
  - 严重过敏判定从直接比较 `severity == 'severe'` 改为调用 `isSevereAllergy(a)`，覆盖 severity 缺失场景。
- 新增 `lib/features/medicine/domain/services/ingredient_canonicalizer.dart`
  - 提取共享 canonical ingredient map（acetaminophen/aspirin/ibuprofen/青霉素/头孢等 15 组）。
  - `canonicalIngredientKeysFor(...)`：把 token 映射为 canonical key，用于重复成分检测。
  - `expandCanonicalIngredientTokens(...)`：展开同组所有变体，用于过敏匹配。
- `lib/features/medicine/domain/entities/medicine_risk_medicine_detail.dart`
  - 新增 `drugbankIds` getter：DrugBank 来源返回 `sourceRefId`，CN 来源返回 `detail.drugbankIds`。
  - 新增 `canonicalIngredientKeys` getter，把提取的成分 token 映射为 canonical key。
  - `allSourceIngredientTokens` 改为基于 canonical keys，便于过敏跨语言匹配。
- `lib/features/medicine/domain/services/medicine_risk_checker.dart`
  - 过敏 finding 的 severity 改为基于 `inferredAllergySeverity`（severe→high，moderate→medium，mild→info，缺失默认 high）。
  - 过敏 token 扩展从本地 `_allergyCrossLanguageMap` 改为使用 `expandCanonicalIngredientTokens`。
  - 重复成分检测从 `normalizedIngredientTokens` 改为 `canonicalIngredientKeys`，实现对乙酰氨基酚 ↔ acetaminophen 等跨语言重复识别。
  - 药物相互作用检测改为对称检查：A 的 `drugbankInteractionTargets` 与 B 的 `drugbankIds` 取交集，反之亦然，从而让有 `drugbankIds` 映射的中文药参与 DrugBank 相互作用检查。
- Regenerated `packages/lucent_openapi`。

**Tests**

- `test/medicine/medicine_risk_checker_test.dart`
  - 新增 `detects DrugBank-CN interaction via mapped drugbankIds`。
  - 更新 `detects CN duplicate ingredient and food interaction` 的证据期望为 canonical key `acetaminophen`。
  - `_detail` helper 增加 `drugbankIds` 参数。
- `test/medicine/red_flag_evaluator_test.dart`
  - 新增 `Rule 1: missing severity with anaphylaxis reaction still triggers red flag`。
