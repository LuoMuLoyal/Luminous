# 2026-06-12

## Report frontend contracts

- wired `Report` to the new Lucent `/api/v1/user/reports/dashboard` contract instead of the old static mock-only provider path
- added `ReportRemoteDataSource` and `LucentReportRepository`, and mapped generated OpenAPI report DTOs into a dynamic frontend report domain model
- removed `ReportCopyKey`-driven fake data modeling so score summary, findings, pattern text, metric values, and trend values now come from the backend contract
- changed Report page behavior to fetch real data on entry for signed-in users, keep skeleton loading during fetch, and show an explicit signed-out gate instead of rendering a fake report as the main path
- kept report export cards as frontend placeholders because export contracts/files are still out of scope for this phase
- updated report widget tests to cover signed-in rendering, loading skeleton behavior, and signed-out gating against the new provider flow

## Report chrome and refresh cleanup

- removed remaining report-layer hardcoded English chrome from repository/entity mapping and moved metric titles, summary mode copy, and export card copy back under frontend l10n ownership
- changed report summary/export domain modeling to enum-driven UI chrome so the data layer only carries dynamic backend content and stable action kinds
- added mobile `RefreshIndicator` as the primary manual refresh path for Report while keeping the explicit sync button as a fallback action
- updated report widget tests to cover pull-to-refresh without relying on duplicated loading copy

## Report signed-out state alignment

- changed Report signed-out behavior from a full-page auth error gate to an in-tab placeholder dashboard with an inline login notice
- aligned Report with the existing `Today / Record / Mine` protected-tab pattern: signed-out root tabs keep their chrome and placeholder sections instead of rendering an error page
- kept protected-data requests blocked while signed out; the signed-out dashboard is local placeholder data only

## OpenAPI generated-noise cleanup

- updated `tool/regenerate_lucent_openapi.dart` to pass generator global properties that disable generated Markdown docs and generated package stub tests
- added a post-generation cleanup step that removes legacy `packages/lucent_openapi/doc/` and `packages/lucent_openapi/test/` directories and strips those entries from `.openapi-generator/FILES`
- kept the generated Dart client itself committed and analyzable; only low-value doc/test noise is being removed
