## Mock 数据 Debug-Mode 门控

### 问题

项目中 5 个 dashboard/workspace provider 在未登录时直接返回 mock 数据。虽然已标记 `[DEMO]`，但 Release 构建中仍会出现假数据。

### 方案

- `kDebugMode` 下保留 mock 数据（开发预览用）
- Release 模式下返回 domain 模型自身的 `signedOut()` 工厂，提供干净的"未登录空态"

### Domain 模型新增 `signedOut()` 工厂

| 模型 | 文件 | 空态内容 |
|------|------|---------|
| `TodayDashboard` | `today_dashboard.dart` | 空体征、零计数、空列表 |
| `RecordDashboard` | `record_dashboard.dart` | 空时间线、空日历标记、标准快速操作/过滤器 |
| `MedicineWorkspace` | `medicine_workspace.dart` | 零剂量、空用药计划、仅搜索快捷操作 |
| `ReportDashboard` | `report_dashboard.dart` | 零评分、空指标/趋势/导出 |
| `MineDashboard` | `mine_dashboard.dart` | 访客档案、零完成度 |

### Provider 门控（5 个文件）

统一模式：
```dart
if (session.isConfirmedSignedOut) {
  if (kDebugMode) return Mock;           // 开发预览
  return XxxDashboard.signedOut();       // 生产空态
}
```

新增 `import 'package:flutter/foundation.dart';` 到所有 5 个 provider。

涉及文件：
- `today_dashboard_provider.dart`
- `record_dashboard_provider.dart`
- `medicine_workspace_provider.dart`
- `report_dashboard_provider.dart`
- `mine_dashboard_provider.dart`

### 未登录提示

Today/Record/Medicine 页面暂未添加显式"请登录"banner（Report/Mine 已有）。
这是在 `kDebugMode` 门控基础上的后续 UX 优化，不影响本次修复范围。
