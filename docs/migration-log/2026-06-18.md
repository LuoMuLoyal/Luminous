# 2026-06-18

## AI Chat 会话持久化首轮打通

- `/assistant` 现在会在已登录状态下调用 Lucent `GET /api/v1/user/ai-chat/latest`，恢复当前用户最新一条持久化对话，而不是每次重新打开都回到空白临时会话。
- AI chat controller 新增会话恢复状态，与 capabilities 加载拆开处理，避免“能力先返回、历史后覆盖”的竞态；会话恢复失败时页面会显示明确错误和重试入口。
- 清空动作不再只是本地 wipe，而是调用 Lucent `POST /api/v1/user/ai-chat/latest/clear` 归档最新 active conversation；清空失败时保留当前消息，不制造前后端状态分叉。
- SSE 最终结果现在会读取 `conversationId`，为后续多会话扩展保留真实会话标识。
- 修复失败重试重复追加用户消息的问题：如果失败前的最后一条就是待重试用户消息，retry 会复用该消息而不是再插入一遍。
- 调整禁用态行为：如果用户已关闭 AI chat 但存在已恢复历史，对话内容仍然可见，只禁用继续发送；只有“无历史且不可发消息”时才显示空禁用占位。
- AI chat widget tests 新增持久化恢复、后端清空、retry 不重复、禁用态保留历史等覆盖。
- `/assistant` 右上角现在固定显示加号入口；点击后会开始一条新对话，当前实现会先归档当前 active conversation，再把页面重置到新的空白会话状态。

## AI Chat 最近会话列表

- `/assistant` 右上角新增历史入口，点击后会打开最近会话列表底部面板，而不是把历史入口继续塞回设置页。
- 最近会话列表现在读取 Lucent 的真实持久化数据，按最近活跃时间展示，而不是本地临时状态。
- 列表点击后会调用后端打开该会话，并把它切换成当前 active conversation；页面随后直接显示对应完整历史。
- 当前会话在最近列表里有明显“当前”标识，新对话的 `+` 行为保持不变，仍然是归档当前 active conversation 后开始空白会话。
- AI chat repository/controller/page/tests 现在都覆盖了 recent-conversation list/open 这一轮最小多会话流转。
