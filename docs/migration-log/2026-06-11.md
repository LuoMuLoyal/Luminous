# Migration Log - 2026-06-11

## Mine And Settings Contracts — Luminous Frontend (Phase 3 & 4)

### Mine Campus Services Remote API

- Replaced static `_campusServices` in `LucentMineRepository` with a live call through `supportResourcesProvider('campus')`, backed by `GET /api/v1/public/support-resources?scope=campus`.
- Added `_mapSupportResource` to convert `SupportResourceDto` → `MineActionEntry`, including string-to-`IconData` mapping (`local_hospital`, `favorite`, `medical_services`, `emergency`, `school`, `support`, etc.) and color routing.
- API failures gracefully fall back to `_fallbackCampusServices` so the Mine page never blocks on a missing network.
- Removed the redundant `.timeout(const Duration(seconds: 5))` from `mine_dashboard_provider.dart`; Dio already enforces its own connect/send/receive timeouts.

### Settings Privacy / AI Toggles

- Extracted the privacy group into `_PrivacySettingsSection`, reading from `userSettingsControllerProvider`.
- Privacy report row now shows `dataSharingConsent` state dynamically (`minePrivacyShareAfterGrant` vs `minePrivacyOnlyMe`) and tapping it toggles consent via `PATCH /api/v1/me/settings`.
- AI summary row now binds its `Switch` to `aiSummariesEnabled` and tapping toggles the value remotely. Unsigned-in users see the auth-required dialog instead of a mute toggle.

### Settings Reminder Summary Reads Local State

- Extended `NotificationSettingsController` with `waterReminders` and `sleepReminders`, persisted via `SharedPreferences` alongside the existing medication/health/weekly keys.
- Extracted the reminders group into `_ReminderSettingsSection`; medicine/water/sleep rows now read `notificationSettingsControllerProvider` and display `mineReminderEnabled` / `mineReminderDisabled` instead of a hardcoded "已开启".

### Data Export Status

- Extracted the advanced group into `_AdvancedSettingsSection`; the export row reads `dataExportControllerProvider` (latest request via `GET /api/v1/me/data-export-requests/latest`).
- Trailing label switches on `DataExportStatus`: `requested`/`processing` → `mineExportStatusPending`, `completed` → `mineExportStatusCompleted`, `failed` → `mineExportStatusFailed`.
- Tapping triggers `POST /api/v1/me/data-export-requests` and shows a toast confirming the request. No file download is performed.

### Help / About Real Pages

- Help row fetches `supportResourcesProvider('help')` and opens a `showModalBottomSheet` listing each resource title/subtitle.
- About row fetches `appInfoProvider` (`GET /api/v1/public/app-info`) and opens a bottom sheet showing app name, version, and description.

### Localization

- Added five new ARB keys:
  - `mineReminderDisabled`
  - `mineExportStatusPending`
  - `mineExportStatusCompleted`
  - `mineExportStatusFailed`
  - `mineExportRequested`
- Regenerated Flutter localizations (`flutter gen-l10n`).

### Post-Review Fixes

- **P1 — Signed-out Settings must not call protected APIs**: `_PrivacySettingsSection` and `_AdvancedSettingsSection` now guard `ref.watch(userSettingsControllerProvider)` and `ref.watch(dataExportControllerProvider)` behind `signedIn`. When signed out the widgets render with default local values (`false` / `null`) and never trigger `GET /api/v1/me/settings` or `GET /api/v1/me/data-export-requests/latest`.
- **P2 — Mine campus resources display backend title/subtitle**: `_ServiceRow` now uses `entry.rawTitle ?? mineCopy(...)` and `entry.rawSubtitle ?? mineCopy(...)` so backend-provided text takes precedence over localization keys. `_mapSupportResource` now maps resource icons to distinct `MineCopyKey` pairs (`campusSupportTitle`, `campusPharmacyTitle`, `campusEmergencyTitle`, `campusHospitalTitle`) instead of hardcoding the hospital key for every entry.
