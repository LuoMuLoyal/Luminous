# Migration Log - 2026-06-10

- Extended Lucent medicine reminder schedules with optional `startDate` and `endDate` fields.
- Added Lucent `UserReminderDelivery` storage plus read-only `GET /api/v1/me/reminder-deliveries?date=&limit=` for future audit rows; no worker writes rows yet.
- Regenerated the Luminous Lucent OpenAPI package from the updated backend contract; generated baseline is now 39 paths / 109 schemas.
- Wired Luminous reminder reads/writes through the new date-window fields while preserving raw Dio writes for nullable reminder behavior.
- Added Medicine reminder create/edit rows for start date, end date, end-date clearing, and date-range validation.
- Added a local reminder sound preference stored on-device with `SharedPreferences`.
- Added a flat reminder method panel showing local notification state, SMS unavailable state, and selected local sound.
- Added reminder detail delivery-history display backed by Lucent's read-only delivery-log API, including a true empty state when no audit rows exist.
- Updated Chinese and English ARB strings and regenerated Flutter localizations.
- Updated reminder data-source and page tests for date windows, local sound, SMS unavailable state, and delivery-history UI.

## Medicine Reminder Local Notifications

- Added a three-layer local notification path for medicine reminders: pure planner, platform gateway, and a Riverpod sync coordinator.
- Added a watch-style sync provider kept alive from `LuminousApp`, so auth restore, reminder list invalidation, notification permission changes, and local sound changes all reschedule automatically without page-level calls.
- Scheduled concrete reminder instances for the next 7 days and capped the pending queue at 60 entries to stay below iOS pending-notification limits.
- Added deterministic per-occurrence notification IDs plus persisted scheduled-ID bookkeeping so resync cancels only medicine reminder notifications instead of calling `cancelAll()`.
- Added dual Android channels for sounding and silent reminder delivery; `gentle` currently maps to the same system-default sound path as `default`.
- Added four localization keys for the fallback local notification title/body and the device channel name/description.
- Added planner and coordinator test coverage, plus app/provider wiring so reminder schedules now trigger real on-device local notifications while push, SMS, and backend delivery workers remain out of scope.
