# 2026-06-22

## Tooling

- Migrated the frontend validation and git-hook toolchain from PowerShell to
  Dart: `tool/run_daily_checks.dart`, `tool/run_fullstack_checks.dart`,
  `tool/install_git_hooks.dart`, and `.githooks/pre-commit` / `pre-push` now
  call Dart directly instead of going through `.ps1` wrappers.
- Tightened the new Dart `pre-commit` hook to check formatting only for staged
  Dart files, so unrelated pre-existing format drift elsewhere in the repo does
  not block focused commits.
- Fixed the full-stack validation entry so `tool/run_fullstack_checks.dart`
  uses Lucent's current
  `pnpm test:runtime:start` / `pnpm test:runtime:stop` entrypoints instead of
  the removed PowerShell helper path.
- Fixed the Lucent Windows test-runtime helpers behind that wrapper so
  `pnpm test:runtime:start` now returns after background startup and
  `pnpm test:runtime:stop` kills the full runtime process tree instead of
  leaving orphaned `nest --watch` children behind.
- Pinned Android integration-test support dependencies in
  `android/build.gradle.kts` so Flutter's SDK-level dynamic `androidx.test`
  versions stop forcing flaky Maven metadata lookups during local full-stack
  runs.
- Updated frontend docs to match the current Lucent full-stack runtime startup
  command.
- Tightened the record full-stack lane so it waits for the create flow to pop
  back to the record page and then finds the created note by real timeline
  title content instead of assuming a fake fixed `recordId`.
- Added stable timeline row index keys so full-stack tests can open the newest
  record on a freshly prepared date without guessing Lucent-generated ids or
  depending on text rendering details.
