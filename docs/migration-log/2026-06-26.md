- 新增 `MedicineSafetyTipResponseDto`。
- 在 `MedicinesController` 新增 `GET /api/v1/medicines/safety-tips`，支持 `exclude` query param 用于相邻去重。
- 在 `MedicinesService` 新增 `getRandomSafetyTips`，根据 `Accept-Language` 返回中文或英文文本，随机取 4 条，并排除上一次返回的 id。
- 在 AdminJS 注册 `MedicineSafetyTip` 资源，支持 CRUD；`category` 提供下拉枚举。
- 新增/更新 `medicines.controller.spec.ts` 与 `medicines.service.spec.ts` 测试。
- 导出更新后的 `docs/openapi.json`。

**Luminous frontend**

- Regenerated `packages/lucent_openapi`。
- 新增 `MedicineSafetyTip` entity。
- 新增 `SafetyTipsRemoteDataSource`。
- 新增 `medicineSafetyTipListProvider`（`AsyncNotifier`）管理当前提示列表与刷新行为。
- 新增 `medicine_safety_tip_style.dart`，按 `category` 映射图标与颜色。
- 重写 `_SafetyTipsSection`：移除硬编码 4 条提示，改为从 provider 加载，支持 shimmer skeleton、错误重试、"换一批"真实刷新。

- 更新 `test/medicine/medicine_page_test.dart` 通过。

**Docs**

- 更新 `Luminous/docs/Current_State.md`：Medicine 行加入后端驱动的安全提示。

## 注册页体验完善：跳转登录、确认密码、验证码倒计时

完善 `RegisterPage` 的创建账号流程，使其符合常规注册页交互预期。

### What changed

**Luminous frontend**

- `lib/features/auth/presentation/pages/register_page.dart`

  - 新增确认密码输入框，并与密码框同时支持右侧小眼睛切换可见/隐藏。
  - 提交前校验确认密码是否为空，并校验两次输入密码是否一致。

- `lib/features/auth/presentation/providers/register_form_provider.dart`
  - 新增 `confirmPassword` 字段与 `validatePasswordMatch`。

- 后端 `VerificationCodeService.send()` 已保留同邮箱 60 秒冷却校验（`ResultCode.VERIFICATION_CODE_COOLDOWN`），前端拦截仅作为 UX 增强；本次未改动后端代码，仅复核测试通过。

## Toast 体验优化与 Auth 表单行内校验

优化全局 Toast 的防重、关闭与刘海避让，并把登录/注册/忘记密码的表单空值/格式/一致性校验从 Toast 改为输入框下方红色提示。

### What changed

**Luminous frontend**

- `lib/core/feedback/app_toast.dart`
  - 新增 `_currentMessage` 实现“阀门”逻辑：同一条消息重复触发时，不重新排队，而是重置倒计时（1.8s）。
  - Toast 顶部距离由 `MediaQuery.viewPaddingOf(context).top` + `AppSpacingTokens.x4l` 计算，避开刘海/灵动岛安全区。
  - 左右边距改为 `AppSpacingTokens.md`，宽度更自适应。
  - 新增右侧关闭按钮（`Icons.close`，16px，黑色），移除 `IgnorePointer` 以支持点击关闭。
  - `_AppToastSurface` 改为 `Row` 布局，文本居中，关闭按钮固定在右侧。
- 新增 `lib/features/auth/presentation/widgets/auth_field_error.dart`
  - `AuthFieldError` 组件：按主题 `surface.error` 显示输入框下方红色提示文本，无错误时返回 `SizedBox.shrink()`。
- `lib/features/auth/presentation/widgets/auth_shell.dart`
  - 导出 `auth_field_error.dart`。
- `lib/features/auth/presentation/pages/login_page.dart`
  - 邮箱、密码、验证码字段分别包裹 `Column` 并追加 `AuthFieldError`。
  - 提交时调用 `LoginFormNotifier.validate(...)`，空值/邮箱格式错误直接以行内错误显示，不再弹 Toast。
  - 发送验证码前调用 `validateEmailOnly(...)`，邮箱为空时显示行内错误。
- `lib/features/auth/presentation/pages/register_page.dart`
  - 所有字段使用 `AuthFieldError`；提交时调用 `RegisterFormNotifier.validate(...)`，校验邮箱、验证码、密码、确认密码及两次密码一致性。
- `lib/features/auth/presentation/pages/forgot_password_page.dart`
  - 同样使用行内错误展示邮箱、验证码、密码、确认密码的校验结果。
- `lib/features/auth/presentation/providers/login_form_provider.dart`
  - 状态新增 `emailError`、`passwordError`、`codeError`。
  - 新增 `validate(...)`、`validateEmailOnly(...)`、`_isValidEmail(...)`。
  - 输入更新时同步清空对应字段错误。
- `lib/features/auth/presentation/providers/register_form_provider.dart`
  - 状态新增字段错误，新增 `validate(...)` 校验邮箱格式与密码一致性。
- `lib/features/auth/presentation/providers/password_reset_provider.dart`
  - 状态新增字段错误，新增 `validate(...)`。
- `lib/l10n/app_zh.arb` / `app_en.arb`
  - 新增 `authEmailRequiredError`、`authEmailInvalidError`、`authCodeRequiredError`、`authPasswordRequiredError`、`authConfirmPasswordRequiredError`、`authPasswordsDoNotMatchError`。
